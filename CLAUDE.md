## Architecture Notes

### Pages & Routing

- Pages are static HTML files in `src/pages/` that Astro passes through as-is
- Each `.html` file automatically becomes a route (e.g., `src/pages/about.html` в†’ `/about/`)
- The sitemap is generated via a JavaScript endpoint (`sitemap-0.xml.js`) that returns XML with 1-hour cache
- 404 handling is configured via `_redirects` to redirect common sitemap probe attempts to `/404.html`

### Deployment Config

- `_headers` (in `public/`) is copied to Cloudflare Pages and defines response headers for all routes
- `_redirects` (in root) handles URL redirects and 404 handling
- **Both files must exist in `public/` or be in the root** for Cloudflare Pages to recognize them during build

### Security Headers

The `_headers` file enforces:
- **CSP**: Restricts script/style/font sources to self + Google Fonts; images from self, data, and https; no external connections
- **HSTS**: 1-year strict transport security with subdomain inclusion
- **X-Frame-Options**: Prevents clickjacking (SAMEORIGIN)
- **X-Content-Type-Options**: Prevents MIME type sniffing
- **Referrer-Policy**: strict-origin-when-cross-origin
- **Permissions-Policy**: Blocks camera, microphone, geolocation, payment
- **CORP/COOP**: Same-site/same-origin restrictions

**Note:** CSP may conflict with inline scripts. If adding JS features, verify the CSP allows them or adjust accordingly.

### Sitemap & SEO

- Sitemap is generated via the `sitemap-0.xml.js` endpoint (not a static file)
- All URLs in sitemap have trailing slashes (`/about/`, `/contact/`, etc.)
- Redirect rules in `_redirects` block common auto-generated sitemap probe attempts (sitemap_index.xml, sitemaps.xml, feed.xml, etc.)

## Before Deploying

1. Verify `_headers` and `_redirects` are in the correct locations:
   - `_headers` should be in `public/` (copied to build output)
   - `_redirects` can be in root or public
2. Build locally with `npm run build` and verify the `/dist/` output includes both files
3. Test the sitemap endpoint: `/sitemap-0.xml` should return valid XML with proper headers
4. Check Cloudflare Pages deployment logs for any file placement issues
